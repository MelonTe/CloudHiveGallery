# 一、后端框架说明

设计如下：

```cmd
├───cmd                          # 项目启动入口（`main.go`）
├───config                       # 配置文件目录
├───internal                     # 业务核心代码，不对外暴露
│   ├───controller               # 控制器层，处理请求和响应
│   ├───middleware               # 中间件，如认证、日志、CORS 等
│   ├───model                    # 数据模型（数据库表结构等）
│   ├───repository               # 数据访问层，数据库操作封装
|   ├───ecode					 # 错误code的定义
|   ├───common					 # 统一结构定义，例如统一xiang'y
│   └───service                  # 业务逻辑层，处理实际业务
├───pkg                          # 通用模块，其他包的工具
│   ├───cache                    # 缓存模块（如 Redis）
│   └───db                       # 数据库连接和初始化
├───router                       # 路由层，配置路由
│   └───v1                       # v1 版本的 API 路由
└───tests                        # 测试代码（单元测试、集成测试）

```
